# 要件定義書

## 概要

AI Tech Blog に記事5本の追加、自動記事生成パイプライン、および記事詳細ページへの関連記事レコメンド機能を実装する。既存24本の記事と差別化した深い技術コンテンツを提供し、ブログの価値を高める。

## 背景

AI Company OS の開発実績が蓄積されており、orchestratorアーキテクチャ、QMOフルサイクル実践、ComfyUI/Remotion連携など、外部に価値提供できる技術知見が多数存在する。これらを体系的な記事として公開することで、技術ブランドを強化する。また、手動記事作成の工数削減のため、GitHubのPR/Issue履歴から自動生成するパイプラインを整備する。関連記事レコメンドにより読者の回遊率向上も図る。

## 機能要件

| ID | 要件 | 優先度 | 備考 |
|----|------|--------|------|
| FR-001 | スラグ `ai-company-os-orchestrator-architecture` の記事を作成する | Must | orchestrator + launchd 連携アーキテクチャの詳細解説 |
| FR-002 | スラグ `qmo-fullcycle-scoring-practice` の記事を作成する | Must | 実際のスコア記録を参照した実践ガイド |
| FR-003 | スラグ `remotion-market-analysis-video` の記事を作成する | Must | Remotion による自動市場分析動画生成パイプライン |
| FR-004 | スラグ `comfyui-flux-ai-novel-illustration` の記事を作成する | Must | ComfyUI + Flux.1 を用いた AI小説イラスト自動生成 |
| FR-005 | スラグ `claude-code-multi-agent-team-operation` の記事を作成する | Must | TeamCreate/SendMessage/タスク管理の実践知見 |
| FR-006 | 全記事に ComfyUI Flux.1-schnell でアイキャッチ画像（1024x576）を生成・設定する | Must | 画像なし記事は禁止 |
| FR-007 | `scripts/generate-article.ts` を作成する | Must | GitHub PR/Issue/コミット履歴から記事ネタ抽出→Claude CLI で本文生成→API投稿 |
| FR-008 | 自動生成パイプラインで ComfyUI によるアイキャッチ画像も自動生成する | Must | Flux.1-schnell 使用 |
| FR-009 | 記事詳細ページの末尾に「関連記事」セクションを追加する | Must | 同カテゴリ＋同タグで関連度計算 |
| FR-010 | 関連記事は最大3件表示する | Must | |

## 非機能要件

| ID | カテゴリ | 要件 | 基準値 |
|----|----------|------|--------|
| NFR-001 | パフォーマンス | 記事一覧ページの表示速度 | 初期ロード 2秒以内（LCP） |
| NFR-002 | パフォーマンス | 関連記事クエリのレスポンスタイム | 100ms 以内 |
| NFR-003 | セキュリティ | 自動生成スクリプトの API 認証 | 環境変数 API_SECRET_KEY による Bearer Token 認証 |
| NFR-004 | セキュリティ | GitHub Token の安全管理 | .env ファイル／環境変数経由のみ。コードへのハードコード禁止 |
| NFR-005 | 可用性 | 自動生成スクリプトの冪等性 | 同一 PR/Issue からの重複記事生成を防ぐ（slug 重複チェック） |
| NFR-006 | 保守性 | 記事本文は Markdown 形式で DB 保存 | 既存スキーマ準拠 |
| NFR-007 | アクセシビリティ | 関連記事セクションの画像に alt テキストを設定 | 全アイキャッチに alt 必須 |

## スコープ

### In（対象範囲）
- 記事5本の執筆・投稿（本文 Markdown + メタデータ + アイキャッチ画像）
- `scripts/generate-article.ts` の新規作成（Node.js/TypeScript）
- 記事詳細ページ（Astro コンポーネント）への関連記事セクション追加
- 関連記事取得 API エンドポイント追加または既存 API の拡張
- アイキャッチ画像の自動生成（ComfyUI API 呼び出し）

### Out（対象外）
- 記事の定期自動実行（cron ジョブ化）— 今回はスクリプト手動実行まで
- 多言語対応
- 関連記事の機械学習ベースレコメンド（今回はカテゴリ＋タグマッチのみ）
- CMS 管理画面の改修
- 既存記事の編集・削除

## リスク評価

| ID | リスク | 影響度 | 発生確率 | 対策 |
|----|--------|--------|----------|------|
| R-001 | ComfyUI が停止していてアイキャッチ画像生成に失敗する | 高 | 中 | 事前に `curl http://localhost:8188/system_stats` でヘルスチェック。失敗時はスクリプトをエラー終了させ、手動対応 |
| R-002 | 自動生成記事のコンテンツ品質が低く公開に適さない | 中 | 中 | 生成後に人手レビュー（tto 承認）を必須とする。API 投稿前に `--dry-run` モードで本文を確認できる仕様とする |
| R-003 | slug の重複により記事投稿が失敗する | 中 | 低 | 投稿前に `GET /api/articles/:slug` で存在確認。重複時はスキップしてログ出力 |
| R-004 | 関連記事クエリが記事数増加とともに遅延する | 低 | 低 | カテゴリ・タグカラムにインデックスを付与。今回5件追加程度では影響なし |
| R-005 | GitHub API レートリミット（5000req/h）に到達する | 低 | 低 | generate-article.ts にキャッシュ機構を追加。1回の実行で取得する PR 数を 50 件に制限 |

## ロールバック方針

- **切り戻し条件**: デプロイ後に関連記事セクションの表示崩れ、または生成記事に不正コンテンツが検出された場合
- **切り戻し手順**:
  1. 不正記事は `DELETE /api/articles/:slug`（管理者 API）で即時削除
  2. 関連記事 UI に問題がある場合は、Astro コンポーネントの該当セクションをコメントアウトして再ビルド・再デプロイ（`docker compose up -d --build frontend`）
  3. `generate-article.ts` は DB 書き込みのみのため、スクリプト削除で即時停止可能
- **影響範囲**: 記事詳細ページ（関連記事セクション）、新規追加記事5本、自動生成スクリプト（独立ファイル）

## 受入基準（AC）

| AC-ID | 条件 | 確認方法 |
|-------|------|----------|
| AC-001 | 5本の記事が `http://localhost:3100/articles/{slug}` でアクセス可能であること | ブラウザでの手動確認、または Playwright E2E テスト |
| AC-002 | 全5本の記事にアイキャッチ画像が設定されており、記事一覧で表示されること | 記事一覧画面で目視確認。DBの `thumbnail_status = 'completed'` であること |
| AC-003 | 記事詳細ページ末尾に「関連記事」セクションが表示され、同カテゴリまたは同タグの記事が最大3件表示されること | 既存記事のうちカテゴリが一致する記事詳細ページを開いて確認 |
| AC-004 | 関連記事が0件の場合にセクション自体を非表示（または「関連記事なし」表示）にすること | カテゴリ・タグが他記事と一致しない記事で確認 |
| AC-005 | `scripts/generate-article.ts` が引数なしで実行でき、直近のGitHub PRから記事案を出力すること | `npx ts-node scripts/generate-article.ts --dry-run` が正常終了し Markdown が stdout に出力されること |
| AC-006 | 自動生成スクリプトが同一 PR から2回実行しても重複記事が作成されないこと | 同一 PR を対象に2回実行し、2回目は "slug already exists" のログが出てスキップされること |
| AC-007 | 記事本文が1500文字以上であること（5本全て） | DB の `content` カラムの文字数確認、または記事ページの読了時間表示（3分以上） |
| AC-008 | アイキャッチ画像サイズが 1024×576px であること | `file` コマンドまたは画像プロパティで確認 |
| AC-009 | `generate-article.ts` が GitHub Token 未設定時に適切なエラーメッセージを出力して終了すること | 環境変数 `GITHUB_TOKEN` を未設定で実行し、エラーメッセージを確認 |
| AC-010 | 関連記事のカードにタイトル・アイキャッチ・カテゴリが表示されること | 記事詳細ページの関連記事セクションで目視確認 |
