# Development Brief — AI Tech Blog

## Mission
**AI Tech Blog - ttoClaw収集ニュースの自動ブログ化**

ttoClaw が Slack チャンネル（#claude-code-news, #sns-trendy-ai-hacks）に投稿する AI ニュース・技術情報を、自動的にブログ記事に変換する Web アプリケーションを構築する。

### Requirements
- ttoClaw の 6 時間毎ニュース収集データをブログ記事化
- ブラウザで閲覧可能（Tailscale 経由）
- Docker Compose で完結
- AI Company OS ダッシュボードから Start/Stop 可能

## Repository

5. **README.md**: Already exists - update with final setup instructions and architecture

## Architecture

```
ttoClaw News Data (JSON/API)
  → News Ingestion Service (fetch, parse, deduplicate)
  → Article Generator (Claude API for summarization & article writing)
  → PostgreSQL (article storage)
  → Blog Reader (Next.js or Astro static site)
  → Caddy (reverse proxy, port 3100)
```

## Tech Stack Recommendations
- **Backend**: Node.js with Express or Fastify
- **Article Generation**: Anthropic Claude API (`@anthropic-ai/sdk`)
- **Frontend**: Astro (lightweight, static-first blog) or Next.js
- **Database**: PostgreSQL 16
- **Reverse Proxy**: Caddy
- **All containerized** with Docker Compose

## Development Process

Follow SDD (Specification-Driven Development) + TDD (Test-Driven Development):

1. Write specification first (API contracts, data models, architecture) - see Issue #3
2. Write tests before implementation
3. Use GitHub Flow: create feature branches from main, open PRs
4. Ensure CI passes (lint, test, build) - GitHub Actions CI is already set up
5. Run security review before merging

## Quality Requirements

- OSS quality: clean, well-documented code
- No secrets in code (use .env, GitHub Secrets)
- Test coverage >80%
- All dependencies pinned to specific versions
- Security: no command injection, XSS, SQL injection
